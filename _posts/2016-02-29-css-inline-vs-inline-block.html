--- title: "CSS Display Property: Using Inline vs. Inline-Block" subtitle: "Understanding the Difference Between Inline and Inline-Block" date: 2016-02-29 categories: css ---

Inline and Block

Inline and inline-block are values you can apply to an HTML element using the CSS display property. To understand the difference between the two you will need to first have a basic understanding of The inline and block values. Inline and block are the standard values for essentailly all HTML elements.

A block-level element is typically a container element. Some examples of block level elements are: div, section, main, and p. Block-level elements occupy the entire space of their parent elements by default and span the entire width of the page. Elements before and after a block-level element will sit above and below it in the page layout.

I'm a "p" (or paragraph) element. I'm a block-level element.

And I'm a "section" element. I'm a block-level element too. We can't sit next to each other. The teacher thinks we're a bad influence on each other.

An inline-level element only occupies the space of the content inside of it and it flows "inline" with text content. Some examples of inline elements are: strong, span, and a. You can't speficy a height and width in CSS for inline-level elments.

I'm a "span" element. I'm inline. I'm an "a" (or anchor) element. I'm inline too and well behaved enough to sit next to my buddy span.

Inline-Block

Inline-block is basically a combination of inline and block. They are basically block-level elements, but you can specify a height and width in CSS for an element displayed with inline-block. They will also allow other elements to sit to the right and left without pushing them to the next row up or down.

I'm a paragraph elment who's been set to display inline-block.
I'm another paragraph set to inline-block. Stop touching me!
